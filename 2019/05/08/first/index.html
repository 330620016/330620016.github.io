第一次搭建个人博客——七小七

May 8, 2019

搭建个人博客的前因

大概一年之前，经由同学介绍，尝试过一次在Linux系统上搭建个人博客，但是由于对Linux操作系统的不熟悉导致半途而废。

到大三下学期各种面试开始，在网上各种取经，发现基本每个公司都很看重你是否有一个个人博客，来分享你的学习经历，心得或者是生活趣事。这也是开源精神的一种体现。

因此决定重新开始使用Hexo和github pages服务搭建一个属于自己的博客。

使用Hexo和github搭建博客的经历

刚开始搭建博客，就是在网上找别人搭建博客的教程。因为之前做实训项目使用过github来进行代码托管，所以提前下载过Git。因此只需要再安装nodejs和Hexo这两个软件。这里就不在赘述具体的教程，附上一个我参考的别人的教程，写的还是比较详细。转自知乎

值得注意的是，由于不同环境下可能存在的差异，一个人的教程并不能解决你的所有问题，而且初次接触很多陌生的词语会让你感觉很头大，因此你需要参考许多教程，我大概参考了4,5篇博客才完成了初步的搭建。

使用Hexo和github搭建博客的原理

其实在刚开始搭建根本不懂什么原理的，就是按着教程走，出现问题就查问题，搭完以后决定搞懂究竟是什么原理，什么是hexo和github pages，为什么用它们可以搭建一个博客呢？

GitHub Pages 是面向用户，组织和项目开放的公共静态页面搭建托管服务，站点可以被免费托管在Github上，你可以选择使用Github Pages默认提供的域名github.io或者自定义域名来发布站点。

我的理解就是你如果想建一个网站，需要去租服务器，购买域名这些操作吧。GitHub Pages帮你省去了这些操作，你可以直接在GitHub创建一个托管仓库，你可以直接把你写好的静态网页托管在这个仓库里。只不过仓库的名字必须是username.github.io。同时这也将成为访问你这个静态网页的域名。

这里的静态网页指的是你实时的更改网页数据并不会直接在这个网站上显示，就像我现在编辑这篇博客，不可能一次性不更改直接写到底吧，也是一边修改一边看效果，我每次保存md文件后，想看效果是通过http://localhost:4000 这个端口。当我觉得没问题了以后，再使用就在 Git Bash 中使用 hexo d 指令，将最新的改动上传至 GitHub 的托管仓库上，这样我才能在这个 GitHub 提供的域名下看到博客内容的更新https://330620016.github.io/。

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

我的理解是Hexo就是别人写好的博客框架，它有许多开源的主题供你在搭建博客时选择，就是省去你自己去写Web前端，当然如果你熟悉Web前端开发你可以更高效地按照自己的审美来设计博客，如果你实在对Web前端了解很少你也可以使用别人的theme通过简单地修改 _config.yml 的配置文件进行博客创作。

在解释Hexo的时候又引入了一个新的名词Markdown。在这里强烈推荐大家学习使用Markdown。

Markdown是一种轻量级标记语言。它允许人们“使用易读易写的纯文本格式编写文档，然后转换成有效的XHTML（或者HTML）文档

Markdown的优点在这里不赘述，附上链接。值得注意的是Github文档默认都是Markdown格式的。

使用Hexo和github搭建博客的感受

这次尝试搭建博客过程还算顺利，说实话其实根本没有任何技术难题，主要是软件配置和使用，以及对配置文件的修改，只要耐心去做，不会的多找找资料很容易就可以完成。

头一次体会到开源世界的便利，什么都不用自己写，别人都封装好了的主题直接拿来用，想要修改主题的内容却又不太懂前端，直接去Github上问作者，基本都会耐心给你解答。

在这里又想到自己搭建博客的好处还有一点是，在自己的地盘可以自由发表言论，不像在知乎，微博上，你觉得没有任何涉及政治，暴力的话题也很有可能莫名被封禁。

今后我会在博客中记录学习，记录生活，还有继续对个人博客的界面进行优化。
